Parkinson Disease Prediction

What is Parkinson's?

Parkinson's disease causes the death of certain brain cells responsible for controlling movement and coordination. It results in tremors and difficulty walking or moving. Neurons use dopamine, a chemical, to control muscle movement. When Parkinson's occurs, the neurons producing dopamine gradually die, making it hard to control movements. The exact cause of this neuronal degeneration is unknown.

Data Analysis and Preparation
Attribute Description:

The UCI dataset provides the following attributes:

  • name: Subject's name and recording number.
  • MDVP:Fo(Hz), MDVP:Fhi(Hz), MDVP:Flo(Hz): Measures of fundamental vocal frequency.
  • MDVP:Jitter, Shimmer, etc.: Variations in frequency and amplitude.
  • NHR, HNR: Noise-to-tone components in voice.
  • status: Health status (1 = Parkinson, 0 = Healthy).
  • RPDE, D2, DFA, spread1, spread2, PPE: Measures of signal dynamics and complexity.
Distributions:
  • Asymmetric positive distribution: MDVP:Fo(Hz), MDVP:Fhi(Hz), MDVP:Flo(Hz), and other jitter and shimmer measures.
  • Normal distribution: HNR, RPDE, D2, DFA, spread1, spread2, PPE.
  • Binary distribution: Status.
Predictive Models
Objective:

Predict whether a subject has Parkinson's based on voice characteristics.

Data Preparation in RapidMiner:
  • Numerical to Binomial: Converted "status" from numeric to categorical for compatibility with Naive Bayes.
  • Set Role: Assigned "status" as the label (target variable).
  • Select Attributes: Removed "name" as it is irrelevant.
  • Normalize (Z-transformation): Normalized data for better precision.
Algorithms Used:

We implemented Naive Bayes and k-Nearest Neighbors (k-NN). A decision tree could also be used.

Why Naive Bayes and kNN?
  • Naive Bayes: A fast probabilistic model that assumes feature independence, ideal for initial interpretations.
  • kNN: Uses similarity to classify data, suitable for improving diagnostic precision.
Naive Bayes Process:
  • Split Data: Divided data into 70% for training and 30% for testing.
  • Naive Bayes: Trained on the 70% split.
  • Apply Model: Applied to the 30% split.
  • Performance: Provided confusion matrix and predictions.

Confusion Matrix Results: The Naive Bayes model achieved 70.69% accuracy. It performed well in identifying positive cases (93% precision) but struggled with negatives (44% precision).

k-NN Process:
  • Implemented using the Multiply operator to separate from Naive Bayes.

Confusion Matrix Results: k-NN achieved 96.55% accuracy, with 97.73% precision for positive cases and 92.86% for negative cases, demonstrating higher reliability.

Conclusions

Both algorithms are effective, but k-NN outperforms Naive Bayes with a 96% accuracy compared to 70%. This suggests that k-NN is better suited for predicting Parkinson's in this dataset. Future efforts could include testing decision trees for further comparison.
